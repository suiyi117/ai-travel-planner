# AeroTravel 页面信息架构重设计（分步向导 + 摘要轨）

日期：2026-07-11

状态：已批准（2026-07-11 头脑风暴）

## 1. 背景

当前 AeroTravel 前端是固定视口工作台：

- `body` / `.app` 使用 `height: 100vh` 与 `overflow: hidden`，**没有页面级滚动**。
- 桌面为左侧（路线设置 / 智能行程 Tab 切换）+ 右侧常驻地图的双栏格；内容只能在各自 `.pane-body` 内滚动。
- 移动端用底部三 Tab（规划 / 日程 / 地图）切换，但仍是「一屏塞满」的密度。

用户反馈：

1. 交互方式不清晰，不知道先做什么、生成后看哪里。
2. 不能像正常网页一样上下滑动，感觉局促。
3. 设置、行程、地图同时争抢空间，主次不分。

同时产品要服务两类场景：**自己规划出行**，以及 **帮客户做行程并复制/导出交付**。既有「复制客户行程 / 导出长图 / 日历 / 我的行程」能力必须保留。

相关后续能力（可编辑行程、自驾工作流）见 `docs/superpowers/specs/2026-07-10-editable-itinerary-self-drive-design.md`。**本设计只解决页面壳与交互节奏**，不实现可编辑行程本体；布局需为日后 Step 3 扩展预留空间，且不与该文档的「结果区编辑模式」硬冲突——本设计将地图改为按需抽屉后，未来编辑模式应在主内容区展开，地图仍按需打开。

## 2. 目标

- 用**分步向导**替代「所有控件堆在一屏」：路线 → 偏好 → 行程结果。
- 恢复**页面级（主内容区）自然滚动**，去掉闷罐式 `100vh` 三栏锁死布局。
- 桌面保留**左侧摘要轨**（步骤导航 + 实时行程摘要），兼顾个人规划与顾问扫视效率。
- **地图默认不占主视觉**；按需以抽屉/全屏层打开。
- **桌面优先**；窄屏可降级使用，不追求与桌面同等精致。
- 保留交付动作、localStorage 行程、现有 API 与 fallback 行为。
- 不引入前端框架、构建工具、数据库或登录。

## 3. 非目标

- 不重做品牌视觉体系（继续暖色 parchment / terracotta）。
- 不改变后端路由契约（`/api/plan`、`/api/search_pois`、transport 等）。
- 不在本设计中实现可编辑行程、自驾路网优化、登录或多端同步。
- 不把地图改回默认常驻双栏主布局。
- 不新增必需付费第三方服务。

## 4. 核心产品决策

### 4.1 布局方案：向导 + 行程摘要轨（方案 B）

| 区域 | 职责 |
|------|------|
| 顶栏 | 品牌；我的行程；交付动作（复制 / 长图 / 日历）。**生成 CTA 不放顶栏**，放在步骤流内。 |
| 左侧摘要轨（桌面） | 三步导航 + 实时摘要（城市序列、总天数、预算、节奏、默认交通等）；**视口内固定/sticky，自身内容过长时可独立滚** |
| 主内容区 | 当前步骤内容；**主要滚动发生在此区域**（文档流高度随内容增长，或主列 `overflow: auto`，二者择一实现，但**禁止**再锁成「三栏同时 `100vh` 闷罐」） |
| 地图抽屉 | 默认关闭；桌面右侧抽屉；窄屏全屏层 |

### 4.2 三步定义

**Step 1 · 路线**

- 城市序列（添加 / 排序 / 删除）
- 每城天数（1–7），总天数派生汇总
- 段级城际交通（第 2 城起）
- 出发日期
- 主按钮：`下一步：偏好 →`
- 进入 Step 2 前校验：至少 1 个城市、天数合法

**Step 2 · 偏好**

- 旅行节奏
- 默认城际交通
- 预算档位
- 偏好补充文本
- 次按钮：`← 上一步`
- 主 CTA：`生成 AI 规划`
- 生成成功（含合法 fallback 结果应用）后**自动进入 Step 3**
- 生成失败时停留 Step 2，展示状态说明

**Step 3 · 行程结果**

- 标题、摘要、质量提示
- 工具：看地图、刷新班次、重新生成
- Day tabs、类型筛选、时间线（复用现有渲染能力）
- 城际交通列表、费用估算、出行贴士
- 交付动作继续在顶栏可用
- 可回 Step 1/2 修改；**表单与已生成行程数据保留**（除非用户再次生成覆盖）

### 4.3 步骤导航规则

- 左轨显示 1 / 2 / 3 及简短说明；当前步高亮。
- **已到达或已完成的步骤可点击跳转**；不可跳到尚未解锁的步骤（例如从未进入过 Step 3 时，不可从 Step 1 直接点 3——除非已有已应用行程，见下）。
- **解锁规则**：
  - Step 1 始终可进入。
  - Step 2 在 Step 1 校验通过后可进入（用户点「下一步」或之后回跳）。
  - Step 3 在至少成功应用过一次行程（AI 或 fallback / 已加载快照）后解锁；boot 若立即 `applyPlan` 示例行程，则 Step 3 解锁，但**首次进入仍默认停在 Step 1**，摘要轨展示示例摘要，避免一上来淹没在结果里。
- 摘要轨字段随 `state` 与表单控件**实时更新**，不依赖生成。

### 4.4 地图按需

**打开条件**

- Step 3 工具栏「看地图」
- 点击时间线中与地点相关的卡片（景点等）

**行为**

- 桌面：自右侧滑入抽屉，宽度约视口 42%–48%，含地图、当前地点详情、「适配路线」
- 窄屏：全屏层
- 关闭：关闭按钮、`Esc`；桌面可选点击遮罩关闭
- 打开时地图 `invalidateSize`（Leaflet），并定位到对应 POI / 当前日路线
- Day 切换时，若抽屉已打开，更新标题与点位

### 4.5 窄屏降级（约 &lt;900px）

- 左轨改为**顶部水平步骤条**
- 摘要压缩为步骤条下的**单行/两行摘要条**
- 主区整页滚动
- 顶栏交付入口可收入「更多」菜单，避免按钮溢出
- 地图使用全屏层
- 不保留当前底部三 Tab 作为主导航（步骤条取代「规划/日程」；地图改为按需层）

### 4.6 与现有能力的映射

| 现有 | 新位置 |
|------|--------|
| 规划 pane 城市/天数/日期 | Step 1 |
| 规划 pane 节奏/交通/预算/兴趣/生成 | Step 2 |
| 结果 pane 时间线/交通/预算/贴士 | Step 3 |
| 地图 pane 常驻 | 地图抽屉 |
| 桌面 workspace tabs | 左轨步骤（语义升级，不再是「同栏 Tab」） |
| 移动端 mobile-nav | 顶部步骤条 + 按需地图 |
| 顶栏生成按钮 | 移除；仅 Step 2 主 CTA（可选：Step 3「重新生成」） |
| 复制/长图/日历/我的行程 | 仍在顶栏（窄屏可进更多菜单） |

## 5. 前端状态与模块

### 5.1 新增 UI 状态（建议）

在现有单一 `state` 或平行 UI 状态中增加：

- `wizardStep`: `1 | 2 | 3`
- `mapDrawerOpen`: `boolean`
- `mapDrawerFocusId` 或等价：当前聚焦行程项 / POI（可空）

不改变行程领域模型字段（`cities`、`itinerary`、`transportGuide` 等）的语义。

### 5.2 模块职责

| 模块 | 变化 |
|------|------|
| `static/index.html` | 重排 DOM：顶栏、摘要轨、三步面板、地图抽屉结构 |
| `static/css/styles.css` | 去掉全页 lock scroll；摘要轨 + 主区布局；抽屉；窄屏步骤条 |
| `static/js/app.js` | 步骤切换、校验、生成后跳转、抽屉开关、键盘 Esc |
| `static/js/app.js` | 摘要轨渲染；步骤相关 UI；时间线点击打开抽屉 |
| `static/js/planning/map.js` | 抽屉打开时初始化/resize/定位；关闭时可不销毁实例以保性能 |
| `static/js/core/state.js` | 如需，导出 wizard/map UI 默认值 |
| 后端 | **无变更** |

### 5.3 行为保留（硬约束）

- `applyPlan()` 仍为结果水合主路径
- boot 仍可构建并应用 fallback/示例行程，保证首屏有内容
- `fetchJson()` 对 `file://` / 非 8000 端口的后端回退逻辑不变
- per-city `days` 为天数真相来源
- 所有 POI 元数据经 `escapeHtml` / `cleanMetaValue` 后再插入 HTML
- 运输段 key 格式 `"A → B"` 稳定

## 6. 错误与加载

- Step 1 校验失败：字段旁或步骤底提示，不进入 Step 2
- 生成中：Step 2 主按钮 loading/disabled；`statusNote` 或等价区域显示进度
- 生成失败且无可用 fallback：留在 Step 2，说明原因
- 生成失败但应用了本地 fallback：进入 Step 3，并明确标注演示/降级来源（沿用现有状态文案风格）
- 地图资源不可用：抽屉内显示现有 empty 提示，时间线仍可用

## 7. 测试与验收

### 7.1 自动化

- 现有 `.\scripts\check.ps1` 全通过
- 若有纯函数（步骤解锁、摘要派生），补 `tests/` 或前端可测工具函数的单元测试；不强制上浏览器 E2E 框架

### 7.2 手工桌面冒烟

1. 打开首页：默认 Step 1，可整页/主区滚动，不感觉三栏闷罐
2. 填两城与天数 → Step 2 → 生成 → 自动 Step 3，时间线可见
3. 左轨摘要与顶栏 route meta 正确
4. 点景点打开地图抽屉并定位；Esc 关闭
5. 回 Step 1 改城市再生成，结果更新
6. 复制 / 导出长图 / 日历 / 我的行程仍可用
7. 控制台无新增 error/warning

### 7.3 窄屏冒烟

1. 步骤条可切换（受解锁规则约束）
2. 主区可滚动
3. 地图全屏层可开关

## 8. 成功标准

- 用户能明确感知「三步流程」，无需在挤在一起的面板间猜测。
- 主内容可滚动，信息不再全部挤压在一屏固定高度内。
- 地图不默认抢占一半桌面宽度。
- 个人规划与客户交付入口都找得到且可用。
- 无框架迁移、无 API 破坏、无持久化模型破坏。

## 9. 实现顺序建议（供 writing-plans 展开）

1. HTML 结构骨架 + CSS 滚动/摘要轨/步骤（可先静态切换）
2. 步骤状态机 + 校验 + 与现有表单控件接线
3. 生成 CTA 迁到 Step 2，生成后进入 Step 3；顶栏生成按钮移除
4. 结果区迁入 Step 3，复用 render 函数
5. 地图抽屉 + Leaflet resize/定位
6. 窄屏降级
7. 冒烟与 check.ps1
