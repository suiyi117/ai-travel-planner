# 人工冒烟检查清单

## 前置条件

- 服务已启动并可访问。
- `.env` 中配置了 `AMAP_KEY` 和 `AI_API_KEY`。
- 使用外置浏览器打开应用首页（不要使用 Codex/ChatGPT 应用内置浏览器）。
- 测试时记录环境、commit、浏览器版本和执行时间。

## P0 API 检查

```powershell
curl http://localhost:8000/api/health
curl "http://localhost:8000/api/city_center?city=北京"
curl "http://localhost:8000/api/search_pois?city=北京&keywords=景点&count=5"
curl "http://localhost:8000/api/weather?city=北京"
```

- [ ] `/api/health` 返回 `status=ok`。
- [ ] `/api/config` 默认不返回 `amap_key` 和 `amap_security_key`。
- [ ] `/api/city_center?city=西安` 返回西安市附近坐标，而不是同名区县。
- [ ] `/api/search_pois` 返回 POI 列表，字段包含名称、地址或坐标。
- [ ] `/api/weather` 失败时也返回 200 与空 `forecasts`，不抛 500。

## P0 浏览器主流程

- [ ] 首页可打开，首屏有可交互的演示行程。
- [ ] 首页默认停在向导 Step 1（路线），主内容可滚动，非三栏闷罐。
- [ ] 左侧（或窄屏顶部）可见步骤与行程摘要。
- [ ] 城市列表可添加、删除、调整每城天数。
- [ ] Step 1 校验通过后进入 Step 2；主 CTA 为生成规划。
- [ ] 选择至少两个城市后可生成行程。
- [ ] 生成期间有加载状态；成功进入 Step 3。
- [ ] 后端未启动或页面不是 `http://localhost:8000` 时，失败提示为「无法连接后端…」并回落到本地演示规划（不应只显示英文 `Failed to fetch`）。
- [ ] 景点数据全部为空时，提示检查高德 Key / 网络，并回落本地演示规划。
- [ ] 生成成功后自动进入 Step 3，时间线/交通/预算可见。
- [ ] 生成结果包含每日时间线、城市天气和交通建议。
- [ ] 地图默认不展示；「看地图」或点击景点后打开抽屉/全屏层，可关闭。
- [ ] 地图抽屉内可缩放、拖动，标记点可点击。
- [ ] 2× 高分屏下地图使用矢量底图，中文地名清晰可读，标记点与道路位置对齐；阻断矢量资源后可回退到高德栅格底图。
- [ ] POI 详情中的评分、地址、电话、开放时间正常显示。
- [ ] 顶栏复制/长图/日历/我的行程可用；窄屏「更多」可访问导出。
- [ ] 顶栏不再放置主「生成规划」按钮（生成在 Step 2）。
- [ ] 浏览器控制台无新增 error。

## P1 交通与预算

- [ ] 北京 -> 西安等跨城路线可刷新火车班次。
- [ ] 火车班次显示车次号、时间、历时、价格、来源。
- [ ] 航班路线在无第三方 key 时使用内置参考数据或明确显示无结果。
- [ ] 预算档位切换后，交通价格估算或推荐偏好有对应变化。
- [ ] 自驾段不会错误调用火车或航班接口。
- [ ] 交通候选方向不反转。
- [ ] 设置“北京游玩 2 天 → 西安游玩 2 天”后，北京到西安查询日期落在 Day 3，而不是出发首日。
- [ ] 零天过境的连续交通段使用同一日期；环线回程和终点过境交通落在最后一个游玩日。
- [ ] 首选高铁/航班在时间线显示门到门窗口，景点和午餐不与进站、安检、出站或进城接驳重叠。
- [ ] 跨城日容量不足时，景点减少并进入“内部检查”的备选提示，不再硬塞进重叠时段。
- [ ] 改选较晚班次后，时间线按新门到门时段排序；若与景点重叠，节点出现冲突徽章且内部检查报错。
- [ ] 有开放时间的景点不会排在闭馆日或闭馆后；无法容纳时明确降为备选。
- [ ] 自驾段在城市中心可定位时显示高德道路距离/时长参考，并提示出发前复核实时路况。

## P1 本地持久化与导出

- [ ] AI 生成成功后自动出现在“我的行程”。
- [ ] 刷新页面后仍可恢复历史行程。
- [ ] 删除历史行程后计数更新。
- [ ] 导出日历生成 `.ics` 文件。
- [ ] 跨夜交通的日历结束时间落在次日。
- [ ] **导出每日行程图**：多天行程下载多张 `DayN-城市.png`（非单张超长图）。
- [ ] **导出总览图**：导出为 1080×1440（3:4）；地图是主视觉，每日摘要最多两个核心地点，预算、提示、二维码和页脚均未裁切。
- [ ] **导出 PDF 备份**：第 1 页为正式封面且不再空白，第 2 页为总览，第 3 页起为每日安排，末页包含费用/交通/提示；文字可选择、链接可点击。
- [ ] **发布专属行程**：下载 `{token}.html`（及可选 JSON）；说明中的访问路径为 `/t/{token}.html`。
- [ ] 专属预览/发布页：右侧费用/提示/天气/扫码为卡片；每日详情为时间轴卡片，非纯文本墙。

## P2 响应式与可用性

- [ ] 桌面宽屏布局无重叠。
- [ ] 平板 1024×1366 下时间线与地图约按 40%/60% 并排，同高、无横向溢出；顶部只显示总览图与更多操作。
- [ ] 手机 390×844 下地图排在时间线之前，日期栏可横向滚动，三项关键数据保持一行，页面无横向溢出。
- [ ] 手机顶部「更多」包含总览图、PDF 备份和复制全文；底部当前地点/下一站/复制/导航操作条不遮挡时间线并避让安全区。
- [ ] 移动端宽度下主要按钮、时间线、地图和详情面板可用，地点卡不遮挡地图版权信息。
- [ ] 键盘 Tab 可到达主要按钮和输入控件。
- [ ] 页面主要文字在按钮、卡片和面板中不溢出。

## P1 可编辑行程与自驾

- [ ] 生成或载入行程后可切换到“编辑”模式。
- [ ] 想去清单可按名称添加地点，并安排到某一天。
- [ ] 地图点选可调用 `/api/reverse_geocode` 并加入草稿。
- [ ] “仅保存”会按当前编辑顺序应用，不自动重排。
- [ ] “智能优化”返回候选差异，可应用或放弃。
- [ ] 编辑任务可在“调整每日安排”与按需出现的“优化自驾路线”之间切换。
- [ ] “移动到其他日期”打开统一对话框；确认、取消、Escape 和焦点恢复均正常。
- [ ] **路线形态**「单程 / 环线」在 Step 1 常显，不依赖先进入自驾。
- [ ] **起点过境**：首城取消「安排当地游玩」后 days=0，点下一步不报 1–7 天；生成后无该城全日游，有首段转场。
- [ ] **环线**：生成后交通含回程段，末日时间线有「环线回程」；切回单程后回程从时间线消失。
- [ ] 过境/环线草稿点「智能优化」不因 `plan_stay`/`days=0` 返回 422。
- [ ] 自驾节点上移/下移后可“重算道路”，顺序保持不变。
- [ ] `/api/transport/driving-route` 在无 key 时返回 estimate，不破坏草稿。
- [ ] 自驾节点超过 20 个时请求被拒绝或前端明确提示。
- [ ] 费用与贴士、城际交通折叠区的展开状态、箭头和提示文案一致。
- [ ] 分段单选可用方向键、Home/End 操作；向导换步后焦点进入新步骤标题。
- [ ] 顶栏导出/更多菜单可用方向键、Home/End、Escape 操作，菜单关闭后焦点可恢复。
- [ ] 地图抽屉打开后 Tab 不会离开抽屉，Escape 关闭并恢复到触发按钮。
