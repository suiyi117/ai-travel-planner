# 地图体验重构计划：让地图成为主画布

## 意图摘要

把当前“窄地图 + 常驻详情栏”的地图抽屉，重构为**按需打开的宽幅地图工作区**。地图始终占据主要视觉面积；地点详情、路线站点和操作控件以覆盖式浮层或底部浏览条出现，不再永久切走一半地图宽度。

本轮只规划前端改造，保持 FastAPI、地图数据、Leaflet/Amap 接口和现有行程状态不变。

> **当前基线：** 原地图重构任务 1–5 已落地在 `feature/ui-interaction-redesign-map`；本次只规划路线视觉增强与地图缩放控件统一，不回退宽幅地图、浮卡和移动端页片结构。

## 现状判断

截图中的问题不是单纯“地图还不够宽”，而是信息架构错误：

- 右侧详情栏长期占据约一半可用宽度，但大部分区域为空白。
- 地图被限制在左半屏，城市道路、景点分布和整日路线缺少横向上下文。
- 顶部工具栏、地图和详情栏是三个割裂的矩形，没有明确的主次关系。
- 地点详情与地图标记弹窗重复展示相近信息，存在两个焦点。
- `static/index.html` 中的内联 Grid 样式覆盖了原有抽屉布局与移动端规则，后续难以维护。

结论：**不继续调整双栏比例；取消常驻右栏。**

## 核心体验决策

### 桌面端：全幅地图工作区

地图打开后使用接近全视口的覆盖层，地图铺满工作区。

```text
┌──────────────────────────────────────────────────────────────┐
│ Day 1 · 西安      [全天路线] [适配路线]              [关闭] │  浮动工具栏
│                                                              │
│                    地图主画布（100%）                         │
│                                                              │
│   ┌──────────────────────┐                                   │
│   │ 09:30 陕西历史博物馆 │                                   │  选中地点浮卡
│   │ 景点 · 2.5 小时      │                                   │
│   │ 查看当天安排 →       │                                   │
│   └──────────────────────┘                                   │
│                                                              │
│ [01 博物馆] [02 城墙] [03 回民街] [04 大雁塔] →              │  底部站点浏览条
└──────────────────────────────────────────────────────────────┘
```

布局规则：

- 地图画布占覆盖层全部剩余空间，不与详情做固定分栏。
- 顶部工具栏悬浮在地图上，使用轻量暖色表面与 ring，不再占整行大块高度。
- 选中地点只显示一张紧凑浮卡；内容超过合理高度时卡片内部滚动。
- 当天站点在底部横向浏览条中展示，点击站点定位地图并更新浮卡。
- 地图标记弹窗只保留“名称 + 时间”的快速确认，完整信息统一由浮卡承载。
- “全天路线 / 当前地点”使用真实的双状态切换：全天路线适配全部点，当前地点聚焦选中标记。

### 移动端：全屏地图 + 底部详情页片

- 地图占满屏幕，不显示桌面浮卡和常驻横向站点条。
- 顶部仅保留返回、日期标题和“适配路线”。
- 选中标记后，底部详情页片进入 peek 状态，显示地点名、时间、类型。
- 上拉/点击页片展开完整详情；下拉或点击地图回到 peek。
- 站点切换放在页片内的横向列表，避免地图上叠过多控件。
- 触控目标不小于 44px；安全区适配 `env(safe-area-inset-*)`。

## 用户任务与关键流程

### 流程 A：查看一天的地理关系

1. 用户在行程结果页点击“看地图”。
2. 地图以当天全部有效坐标点适配视野。
3. 底部站点条展示当天顺序，当前行程项高亮。
4. 用户平移、缩放地图时，浮卡不遮挡主要路线区域。

### 流程 B：查看一个地点

1. 用户点击地图标记、底部站点或行程时间线卡片。
2. 地图平滑聚焦该地点，但不强制缩放到失去周边上下文。
3. 浮卡/底部页片更新为该地点详情。
4. 用户可继续切换前后站点，不需要关闭地图。

### 流程 C：恢复全天路线

1. 用户点击“全天路线”或“适配路线”。
2. 地图恢复当天全部有效坐标点的边界。
3. 当前地点仍保留选中状态，但不再独占视野。

## 组件与状态

| 模块 | 职责 | 关键状态 |
|---|---|---|
| Map Workspace | 覆盖层、焦点管理、滚动锁定 | closed / opening / open / closing |
| Map Canvas | 地图、路线、标记 | loading / ready / empty / provider-error |
| Floating Toolbar | 日期、视野模式、关闭 | route-view / place-view |
| Place Card | 当前地点详情 | hidden / selected / long-content |
| Stop Rail | 当天站点顺序与切换 | active / overflow / no-coordinates |
| Mobile Sheet | 移动端地点详情 | hidden / peek / expanded |

统一状态来源继续使用现有：

- `state.mapDrawerOpen`
- `state.activeItemId`
- `state.mapFocusItemId`
- `state.currentDay`
- `activeItems()` 与现有行程数据

不新增第二套地图选择状态。

## 交互规则

- 打开地图时保存触发按钮；关闭后恢复焦点。
- `Escape`、关闭按钮和背景退出行为保持一致。
- 地图工作区打开时锁定页面滚动，但地图自身可正常缩放和平移。
- 标记、站点条和时间线三处选择必须同步到同一个 `activeItemId`。
- 无坐标的交通/说明项可出现在站点条，但显示“无地图位置”，不创建伪坐标。
- 连续快速点选只保留最后一次聚焦，不叠加动画或延迟任务。
- 遵守 `prefers-reduced-motion`；地图视野变化可即时完成。
- 详情信息继续经过 `escapeHtml()` / `cleanMetaValue()`，不改变现有安全边界。

## 视觉规则

- 继续使用现有 parchment / terracotta 设计系统，不换肤。
- 地图是唯一大面积内容；浮层使用 Ivory 表面、暖色 ring 和极轻阴影。
- Terracotta 只用于当前站点或一个主动作，不同时铺满按钮、标记和卡片。
- 地图控件与应用控件在圆角、字体和高度上统一；不保留浏览器默认大按钮感。
- 地点标题使用 serif，操作与元信息使用 sans；避免巨大空白详情面板。
- 不新增平台切换器、演示设置或设计说明控件。

## 本轮增量：路线视觉层级与地图控件统一

### 来自当前截图的判断

- 目前路线是一条均匀的橙色虚线，叠在高德底图的道路颜色上，容易被看成“临时估算线”，路线顺序、起终点和当前段都不够明确。
- 路线、编号标记、当前地点卡片和按钮都在争用相近的赤陶色，视觉焦点没有收敛到“今天怎么走”。
- Leaflet 默认的 `+ / −` 控件仍是地图插件的白色方块语气，与应用中的 Ivory 表面、暖色 ring、圆角和字体不属于同一组件家族。

### 目标

- 路线成为地图内第二视觉主轴：打开地图后，用户能在 1 秒内看懂“按什么顺序走、当前在哪、路线是否跨城”。
- 让路线看起来像经过设计的产品信息层，而不是直接把一条 polyline 叠在底图上。
- 让缩放控件看起来像 AeroTravel 的应用控件，同时保留 Leaflet 原生键盘、触控和 ARIA 语义。

### 路线视觉方案

1. **双层路线线条。** 用低透明度的暖色底衬线先把路线从底图中托出来，再叠一条更窄的前景线；两层均使用圆角端点和圆角连接，避免锯齿和廉价虚线感。
2. **按状态控制节奏，不堆彩色。** 真实/已确认路线使用 `--accent` 前景线；城际段使用现有暖色中性层级区分；估算路线保留虚线，但降低装饰感、提高可读性。实现时从现有 CSS token 读取颜色，不在 `static/js/planning/map.js` 新增孤立硬编码色值。
3. **用节点编号表达方向。** 保留站点编号，但让非当前节点更克制，当前节点使用唯一的赤陶高亮；起点/终点通过文字或形状语义区分，不额外铺一层大标签。
4. **只在需要时显示路线说明。** 当一天内存在多种路线状态时，在地图工具层增加一条紧凑图例；单一路线状态不显示图例，避免地图变成说明面板。
5. **控制视觉焦点。** 路线获得主 accent 后，节点条和地点浮卡改用暖色 ring 与中性表面承载，避免路线、标记、按钮同时高饱和。

### 地图缩放控件方案

- 继续使用 Leaflet 原生 `zoomControl` 结构，不引入新的图标库或自绘 SVG；只重做外观和定位。
- 保留左下角的使用语义；如果当前运行时配置仍是 `bottomright`，实现时调整为 `bottomleft`，并避开底部站点条、移动端页片和安全区。
- 外层使用 Ivory 表面、`--border` ring、`--radius-md` 和轻量 elevation，两个按钮组成一个连续的分段控件，不再显示两个孤立白色方块。
- `+ / −` 使用现有 sans 字体与克制字号；hover 使用 Warm Sand，按下态使用 ring，键盘焦点使用现有 `--focus-ring`。
- 每个按钮保持至少 44×44px 的触控面积，保留 Leaflet 的 `aria-label`、Tab 顺序和禁用态；控件层级高于地图瓦片、低于地点浮卡，不能挡住路线节点。
- 桌面、平板、360–430px 移动端均不与底部站点条或详情页片重叠；`prefers-reduced-motion` 下不增加额外控件动画。

## 响应式布局

| 宽度 | 呈现方式 | 地图最低可视面积 |
|---|---|---|
| ≥ 1280px | 视口内宽幅覆盖层；浮卡 + 底部站点条 | 宽度 ≥ 80vw，高度 ≥ 78vh |
| 768–1279px | 全屏覆盖层；浮卡收窄；站点条保留 | 宽度 100vw，高度 100dvh |
| ≤ 767px | 全屏地图 + 底部详情页片 | 地图在 peek 状态可见高度 ≥ 65dvh |
| 360–430px | 紧凑工具栏；操作收敛；安全区适配 | 无横向滚动 |

## 实施任务

### 任务 1：撤销错误的常驻双栏结构

**描述：** 移除 `static/index.html` 中地图面板的内联 Grid、固定 1040px 宽度和详情栏左边框；恢复语义清晰、样式集中管理的结构。

- [ ] HTML 中不再存在地图布局内联样式。
- [ ] 地图详情不再作为固定右栏占据一列。
- [ ] 原有按钮、ARIA 属性和地图容器 ID 保持可用。

**依赖：** 无。

**可能涉及：** `static/index.html`、`static/css/styles.css`。

### 任务 2：建立桌面宽幅地图工作区

**描述：** 用现有覆盖层改成地图全铺底，加入悬浮工具栏、地点浮卡和底部站点条；不改地图数据接口。

**验收标准：**

- [ ] 1440px 视口下地图宽度至少占 80vw。

## 风险与缓解

| 风险 | 影响 | 缓解方式 |
|---|---|---|
| 覆盖层尺寸变化后 Leaflet 瓦片空白 | 高 | 打开、断点变化和页片状态变化后统一调用 `invalidateSize()` |
| 浮卡遮挡当前标记或路线 | 中 | 根据选中点屏幕位置选择左/右下角；保持可关闭/收起 |
| 站点条与地图标记状态漂移 | 高 | 只使用 `activeItemId`，不建立独立 rail 选择状态 |
| 移动端页片手势与地图拖拽冲突 | 中 | 首版以明确按钮/点击切换为主，拖拽手势作为后续增强 |
| 当前工作树有其他未提交修改 | 高 | 继续在 `feature/ui-interaction-redesign-map` 上实现，只处理地图相关前端 diff，不覆盖既有改动 |

## 明确不做

- 不改后端路由、地图供应商或 Amap/Leaflet 数据协议。
- 不把地图变成常驻 100vh 三栏工作台。
- 不新增数据库、登录或持久化结构。
- 不在本轮加入地图搜索、导航规划或新的付费地图能力。
- 不用更大的常驻右侧详情栏替代当前问题。
