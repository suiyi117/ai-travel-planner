## 工作台怎么用

1. 在 AeroTravel 完成路线、偏好并生成/编辑行程。
2. 需要精选外链时：进入编辑模式，点地点 ✎：
   - 先改名称；
   - 再按提示添加最多 **3** 条参考链接，格式 `标签|https://...`（仅 `http`/`https`）。
3. 打开顶部 **导出** 菜单：
   - **参考链接检查**：按天列出地点链接覆盖，输入序号可继续编辑。
   - **预览专属页**：浏览器打开只读客户页预览。
   - **发布专属行程**：先填写访问链接与建议保留日期，再下载自包含 `{token}.html` + JSON，并复制发布说明与客户文案。
   - **导出总览图**：竖向 3:4 风格总览 PNG（逻辑画布 540×720，2× 导出为 1080×1440），用于私信/相册/小红书；地图保持主视觉，每天只保留两个核心地点。
   - **导出 PDF 备份**：打开打印页，另存为 PDF；第 1 页为正式封面，第 2 页为行程总览，第 3 页起为每日时间线，末页集中费用、交通与提示。
   - **复制客户行程 / 导出每日行程图 / 导出日历**：按天导出 PNG，避免整段长图难读。

## 发布前准备

1. 先确定静态托管域名与路径规则，例如：`https://你的域名/t/{token}.html`。
2. 点击 **发布专属行程** 时：
   - **访问链接**：填写客户最终可打开的 URL（用于二维码）。可先填预期地址；若上传后路径不同，改 URL 后重新发布一次。
   - **建议保留至**：可选，默认约为行程结束后 30 天；仅展示在页面上，**不会自动删除**。
3. 浏览器会记住上次使用的域名，方便下次预填。
4. 导出总览图 / PDF / 发布前，工作台会尝试请求后端生成 **高德静态总览图** 并写入 package：
   - 需要服务端配置 `AMAP_KEY`；
   - 失败时**仍可交付**，图上显示「地图见专属链接」占位；
   - 客户 HTML **不会**包含地图 Key；互动地图仍在线加载 Leaflet 瓦片。

## 人工发布（第一阶段）

1. 将下载的 `{token}.html` 上传到静态托管的 `/t/` 目录，最终访问路径为 `/t/{token}.html`（与发布时填写的访问链接 / 二维码一致）。
2. 建议 URL：`https://你的域名/t/{token}.html`。若托管只支持目录形式，可用 `/t/{token}/index.html` 并相应修改填写的访问链接。
3. 页面已带 `noindex`，token 为高强度随机串；勿写入证件号、完整订单号、手机号。
4. 向客户同时发送：专属链接 + 可复制文字 + PDF 备份 +（可选）每日行程图。
5. 需要改行程时：在工作台改结构化行程后重新发布（覆盖同路径或更换新 token）。
6. 过期处理：按「建议保留至」人工删除托管文件即可。

## 客户打开时注意

- 微信 / QQ 等内置浏览器可能无法调起高德 App；页面会提示复制地址，或用系统浏览器打开。
- 高德跳转使用官方 HTTPS URI；不承诺多日路线一键导入。
- 有自驾道路数据时，客户页地图用实线标「道路数据」；否则用虚线标「估算连线」。
- 地点上配置的参考链接会显示为可点击 chips（小红书/网页等）。
- 平板竖屏保留时间线与地图并排；手机端按「行程摘要 → 日期 → 地图 → 关键数据 → 时间线」单列展示，导出操作收在顶部「更多」。
- 手机底部操作条固定显示当前地点、下一站、复制当日和高德导航，并预留系统安全区。

## PDF 打印建议

1. 使用 Chrome / Edge 打开打印对话框并选择 **另存为 PDF**。
2. 纸张选择 **A4**、方向选择 **纵向**、缩放保持默认或 100%。
3. 建议关闭浏览器自带页眉和页脚，开启背景图形，以保留封面路线纹理与日颜色。
4. 导出后快速检查：第一页不是空白；封面、总览、每日页顺序正确；地图已加载或明确显示占位；外部链接仍可点击。

## 技术边界

- 不引入账号、数据库、在线支付。
- 总览地图编号与每日锚点来自同一份 `trip package`。
- 估算连线在客户页用虚线表示，不伪装成真实道路。
- `share_url` / `valid_until` / `route_lines` / `static_map` / `item.refs` 均写入 package 后再生成 HTML / PNG / PDF。
- 静态图仅在工作台预渲染；客户页不调用静态图 API。
- PDF 使用独立打印结构但继续读取同一份 `trip package`；封面路线节点、总览摘要、每日路线和预算提示不会维护第二份内容数据。
